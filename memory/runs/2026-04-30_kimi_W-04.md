## Goal

Alle Frontend-UI-Strings in `code/frontend/src/lib/i18n.ts` zentralisieren. `t()`-Funktion mit Fallback (missing key → console.warn + key return). Alle UI-Strings in den owned Components durch `t()` ersetzen. Kein Englisch mehr im UI — Code bleibt Englisch.

## Done

- `code/frontend/src/lib/i18n.ts`:
  - `t()` akzeptiert nun `string` (nicht nur `I18nKey`) für Fallback-Support.
  - Fallback: `console.warn("missing-i18n-key", key)` + Rückgabe des raw keys.
  - ~80 Keys im `de`-Dictionary (bestehende + neue aus den Components).
  - Englische Labels aus dem alten Dictionary ins Deutsche übersetzt (z.B. "Research only" → "Nur Forschung", "Export Snapshot" → "Snapshot exportieren", "Case" → "Fall").

- `code/frontend/src/lib/i18n.test.ts` (3 Tests):
  1. Bekannter Key wird korrekt aufgelöst.
  2. `{var}`-Replacement funktioniert (z.B. `tour.progress`).
  3. Fehlender Key triggert `console.warn` und gibt den Key zurück.

- Components refactored (alle Roh-Strings → `t()`):
  - `AiPanel.tsx` — 22 String-Replacements (focus labels, trust, badges, buttons, disclaimer).
  - `ConfidenceBadge.tsx` — 8 String-Replacements (bucket labels, tooltips, warning).
  - `DecisionForm.tsx` — 28 String-Replacements (verdicts, confidence levels, override reasons, labels, buttons, states).
  - `FreeTextField.tsx` — 6 String-Replacements (label, optional, checking, PII warning).
  - `DicomViewer.tsx` — 25 String-Replacements (banner, presets, heatmap toggle, info overlays, opacity labels).
  - `App.tsx` — 18 String-Replacements (tabs, sidebar, download button, research mode notice).
