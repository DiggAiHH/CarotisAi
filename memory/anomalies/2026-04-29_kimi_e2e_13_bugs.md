# 13 Bugs aus der K-01..K-16 E2E-Verifikation

> **Wichtig für jedes künftige Modell:** Diese Bugs sind **gelöst**. Wenn du Code in `code/` generierst, prüfe zuerst, dass du keine dieser Anti-Patterns wiederholst.

## B-01 — `requirements.txt` Versions-Pinning bricht Python 3.13

**Symptom:** `pip install -r backend/requirements.txt` schlägt auf Python 3.13 fehl wegen `onnxruntime`/`numpy`/`scipy`-Wheels.

**Fix:** Versionen relaxieren, nicht hart pinnen:
```
onnxruntime>=1.20.0,<1.22
numpy>=1.26,<2.2
scipy>=1.14,<1.15
```

**Lehre:** Bei ML-Stack-Files lieber Range-Constraints als `==` für die Compute-Libs. Hart pinnen nur wenn ein Bug-Workaround nötig ist.

## B-02 — `jsonschema` fehlt in `requirements.txt`

**Symptom:** `decision_tree_service.capture()` wirft `ModuleNotFoundError: No module named 'jsonschema'` zur Laufzeit.

**Fix:** `jsonschema>=4.21,<5.0` zu `backend/requirements.txt` hinzufügen.

**Lehre:** Bei jeder Service-Klasse die externe Libs nutzt: prüfe **beim Schreiben** ob die Lib in requirements ist. Nicht erst wenn der Test fehlt.

## B-03 — `Depends` fehlt im Import in Router

**Symptom:** `code/backend/app/api/routes/health.py` benutzt `Depends(verify_api_key)` aber importiert es nicht — `NameError` beim App-Boot.

**Fix:** Import-Block am Anfang von Router-Files:
```python
from fastapi import APIRouter, Depends, Header, HTTPException, status, UploadFile, File
```
Auch wenn der Router gerade nur `Depends` braucht — Pattern aus FastAPI-Doku copy-pasten.

**Lehre:** Beim Generieren von FastAPI-Routern: ALLE FastAPI-Symbole im ersten Import-Statement, nicht erst wenn nötig.

---

## B-04 — `from __future__ import annotations` bricht `UploadFile` ForwardRef

**Symptom:** `code/backend/app/api/routes/inference.py` mit `from __future__ import annotations` macht `UploadFile = File(...)` zu einem ForwardRef-String — FastAPI scheitert beim OpenAPI-Schema.

**Fix:** **Genau in Router-Files mit `UploadFile`-Parameter** das `from __future__ import annotations` weglassen. Andere Files bleiben wie sie sind.

**Lehre:** `from __future__ import annotations` ist ein Default-Pattern in CLAUDE.md, aber FastAPI's Runtime-Type-Inspection für Multipart-Upload bricht damit. Ausnahme dokumentiert.

## B-05 — Dynamic Import von `scripts/anonymize.py` mit `@dataclass` braucht `sys.modules`-Registrierung

**Symptom:** `anonymization_service.py` importiert `scripts/anonymize.py` dynamisch via `importlib.util.spec_from_file_location()`, aber `@dataclass` schlägt mit `KeyError: '__name__'` fehl, weil das Modul nicht in `sys.modules` steht.

**Fix:**
```python
spec = importlib.util.spec_from_file_location("anonymize", path)
_anon_module = importlib.util.module_from_spec(spec)
sys.modules["anonymize"] = _anon_module    # ⬅ MUSS vor exec_module
spec.loader.exec_module(_anon_module)
```

**Lehre:** Dynamic Imports von Files mit `@dataclass` brauchen `sys.modules`-Registrierung. Bekannter Python-Bug, gut dokumentiert in PEP 451.

## B-06 — `_rate_limit_exceeded_handler` muss explizit definiert werden

**Symptom:** SlowAPI's eingebauter `_rate_limit_exceeded_handler` funktioniert in FastAPI 0.115+ nicht mehr direkt als `app.add_exception_handler` — Deprecation-Warning bei Boot.

**Fix:** Eigenen Handler definieren:
```python
async def _rate_limit_exceeded_handler(request: Request, exc: RateLimitExceeded):
    return JSONResponse(
        status_code=429,
        content={"detail": f"Rate limit exceeded: {exc.detail}"},
    )
app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)
```

**Lehre:** SlowAPI-Defaults nicht blind übernehmen — FastAPI 0.115+ braucht eigene Implementation.

## B-07 — `model_version` / `model_sha` triggert Pydantic v2 Namespace-Warning

**Symptom:** Pydantic v2 reserviert `model_*` Prefix → Warning bei Settings-Klasse mit `model_version`/`model_sha` Feldern.

**Fix:** In `BaseSettings`-Subclass:
```python
model_config = SettingsConfigDict(
    env_file=".env",
    env_file_encoding="utf-8",
    extra="ignore",
    case_sensitive=False,
    protected_namespaces=("settings_",),  # ⬅ entkoppelt model_*
)
```

**Lehre:** Bei Settings-Klassen mit Feldern die `model_` als Prefix haben: `protected_namespaces` überschreiben. Nicht den Field-Name umbenennen.

## B-08 — `settings` beim Modul-Import cachen bricht Tests

**Symptom:** `dependencies.py` macht `settings = get_settings()` auf Modul-Ebene → in Tests werden `monkeypatch.setenv()`-Werte ignoriert, weil das Settings-Objekt schon gecached ist.

**Fix:** **Niemals** Settings beim Modul-Import dereferenzieren. In jeder Funktion frisch holen:
```python
async def verify_api_key(x_api_key: str = Header(...)):
    settings = get_settings()    # ⬅ inline, nicht modul-global
    if x_api_key != settings.api_key:
        raise HTTPException(401)
```

**Lehre:** `@lru_cache` auf `get_settings` ist OK — aber den Aufruf erst im Funktions-Scope machen. Tests können dann `get_settings.cache_clear()` rufen und env überschreiben.

## B-09 — `SQLEnum` ohne `values_callable` speichert Enum-Names statt Values

**Symptom:** `agreement_verdict` als `Enum` wird in SQLite als `FULL_AGREEMENT` (Name) statt `full_agreement` (Value) gespeichert. Kollidiert mit Schema-Validation.

**Fix:**
```python
class AgreementVerdict(str, Enum):
    FULL_AGREEMENT = "full_agreement"
    PARTIAL_AGREEMENT = "partial_agreement"
    DISAGREEMENT = "disagreement"

verdict: Mapped[AgreementVerdict] = mapped_column(
    SQLEnum(AgreementVerdict, values_callable=lambda x: [e.value for e in x])
)
```

**Lehre:** SQLAlchemy's `SQLEnum`-Default speichert Names. Bei String-basierten Enums **immer** `values_callable` setzen.

## B-10 — `test_smoke.py` API_KEY zu kurz, alte Router-Pfade

**Symptom:** Smoke-Tests schlugen fehl mit 401 (zu kurzer Key) und 404 (Pfad nicht da).

**Fix:**
- API_KEY in Tests: 32+ Zeichen (matched config validator)
- Router-Pfade aktualisiert auf aktuelle Struktur (`/health`, `/inference/predict`, `/decision-tree/capture`, `/audit/trail`)
- `InferenceService` mit `AsyncMock` gemockt — kein echtes ONNX-Loading in Smoke-Tests

**Lehre:** Tests sind Living Documents. Wenn Router-Pfade sich verschieben (z.B. künftiges `/api/v1/`-Prefix in K-17), Tests gleichzeitig anpassen.

## B-11 — `conftest.py` ohne `drop_all` vor `create_all` → DB-Leak zwischen Tests

**Symptom:** Test 1 erstellt einen `DecisionTree`, Test 2 sieht ihn noch da, weil in-memory SQLite über Module-Scope geshared wird.

**Fix:** In `conftest.py` Fixture für DB:
```python
@pytest_asyncio.fixture
async def test_db():
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.drop_all)     # ⬅ erst drop
        await conn.run_sync(Base.metadata.create_all)
    yield
```

**Lehre:** Bei Test-Fixtures mit shared Engine: **erst drop, dann create**. Sonst Lese-Verschmutzung zwischen Tests.

## B-12 — Schema zu strikt: `reasoning` und `confidence_self_reported` müssen `null` erlauben

**Symptom:** Decision-Trees aus echtem Klinik-Workflow haben oft kein `reasoning` (Arzt skipt das Feld) und kein `confidence_self_reported`. Strict Schema rejects diese Trees.

**Fix:** In `schemas/decision_tree.schema.json`:
- `reasoning`: `"type": ["object", "null"]`
- `confidence_self_reported`: `"type": ["string", "null"], "enum": [..., null]`

**Lehre:** Pflichtfelder sind teuer. Wenn ein Feld in `05_DECISION_TREE_HARVESTING.md` als „optional" beschrieben ist, **muss** das Schema das auch erlauben — sonst rejecten wir genau die Cases die wir erfassen wollten. Diese Schema-Lockerung wurde bereits durch User vorgenommen.

## B-13 — Unicode-Pfeile (→) brechen auf Windows-cp1252-Console

**Symptom:** `python scripts/generate_demo_model.py` auf Windows-PowerShell-Console → `UnicodeEncodeError` bei `print(' input → ...')`.

**Fix:** ASCII-Pfeile statt Unicode in jedem Skript das auf Windows läuft:
```python
print("Input -> output")    # nicht: "Input → output"
```

**Lehre:** Console-Output in `scripts/*.py` immer ASCII. Markdown-Dokumente dürfen Unicode haben (VS Code rendert UTF-8 korrekt). Trennlinie zwischen "läuft auf Console" vs "läuft als File".

## Top-3 Anti-Patterns die in K-17+ vermieden werden müssen

1. **Niemals** Settings beim Modul-Import cachen (B-08).
2. **Niemals** Unicode-Pfeile in Python-Console-Skripten (B-13).
3. **Niemals** `from __future__ import annotations` in FastAPI-Routes mit `UploadFile`-Parametern (B-04).
